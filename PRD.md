## 1. 개요 (Overview)

본 문서는 대한민국 공인중개사를 대상으로, **슬랙(Slack) 채널에서 명령어 한 줄로 등기부등본과 건축물대장을 자동 조회·출력하는 내부 도구**에 대한 제품 요구사항을 정의한다.

공인중개사는 매물 확인, 계약 준비, 고객 응대 과정에서 등기부등본과 건축물대장을 수시로 열람해야 한다. 현재는 대법원 인터넷등기소, 정부24(세움터) 등의 웹사이트에 직접 접속하여 수작업으로 조회·발급하고 있으며, 이 과정에서 상당한 시간과 반복 노동이 발생한다. 본 도구는 이 워크플로우를 슬랙 봇 기반으로 자동화하여 **업무 효율을 극대화**하는 것을 목표로 한다.

## 2. 배경 및 문제 정의 (Background & Problem Statement)

### 2.1 현재 상황 (As-Is)

공인중개사가 등기부등본 1건을 발급받기 위해 거쳐야 하는 절차는 다음과 같다.

1. 대법원 인터넷등기소(http://www.iros.go.kr) 접속
2. 공인인증서/공동인증서 로그인
3. 부동산 소재지 검색 (시·군·구 → 읍·면·동 → 번지)
4. 해당 물건의 표제부(동·호수) 선택
5. 등기부등본 유형 선택 (전부/갑구/을구)
6. 결제 (건당 700원~1,000원)
7. PDF 다운로드 후 팀원에게 공유

건축물대장 역시 정부24 또는 세움터를 통해 유사한 다단계 절차를 거쳐야 한다. 하루에 수십 건을 처리하는 중개사무소의 경우 이 반복 작업에만 **1~2시간 이상**이 소요된다.

### 2.2 핵심 문제 (Pain Points)

- **시간 소모**: 건당 3~5분 소요, 대량 처리 시 누적 시간이 매우 큼
- **컨텍스트 스위칭**: 슬랙에서 업무 협의 중 외부 사이트로 이동해야 함
- **공유 번거로움**: 발급 후 별도로 파일을 다운로드하여 슬랙/카카오톡 등으로 재전송해야 함
- **반복 입력**: 동일한 주소를 여러 사이트에 중복 입력해야 함

### 2.3 기회 (Opportunity)

슬랙을 이미 업무 메신저로 사용하는 중개사무소에서, 채팅창을 떠나지 않고 명령어 한 줄로 등기부등본·건축물대장을 즉시 확보할 수 있다면, 건당 처리 시간을 **3~5분 → 30초 이내**로 단축할 수 있다.

## 5. 기능 요구사항 (Functional Requirements)

### 5.1 핵심 기능 (Must-Have / P0)

#### F1. 슬랙 슬래시 명령어를 통한 문서 조회 요청

- 사용자는 지정된 슬랙 채널에서 슬래시 명령어를 입력하여 조회를 요청한다.
- **명령어 형식 예시:**
  ```
  /등기 서울시 중랑구 중화동 450 중화한신아파트 103동 904호
  /건축물 서울시 중랑구 중화동 450 중화한신아파트 103동 904호
  /전체 서울시 중랑구 중화동 450 중화한신아파트 103동 904호
  ```
- `/등기` — 등기부등본만 조회
- `/건축물` — 건축물대장만 조회
- `/전체` — 등기부등본 + 건축물대장 동시 조회

#### F2. 주소 및 표제부 파싱

- 입력된 자연어 주소를 파싱하여 **시/도, 시/군/구, 읍/면/동, 번지, 건물명, 동, 호** 단위로 분리한다.
- 지번 주소 및 도로명 주소 모두 지원한다.
- 파싱 실패 시 사용자에게 형식 안내 메시지를 반환한다.

#### F3. 등기부등본 자동 조회 및 발급

- 파싱된 주소 정보를 기반으로 대법원 인터넷등기소 API(또는 연동 가능한 서드파티 API)를 호출한다.
- 등기부등본 전부(표제부 + 갑구 + 을구)를 PDF 형식으로 발급받는다.
- 결제는 사전 등록된 법인/사무소 결제 수단으로 자동 처리한다.

#### F4. 건축물대장 자동 조회 및 발급

- 파싱된 주소 정보를 기반으로 정부24 공공데이터 API(건축물대장 정보 서비스)를 호출한다.
- 건축물대장(일반/총괄표제부)을 PDF 형식으로 발급받는다.

#### F5. 슬랙 채널 자동 업로드

- 발급 완료된 PDF 파일을 요청이 발생한 슬랙 채널에 봇이 자동 업로드한다.
- 업로드 시 다음 정보를 메시지로 함께 제공한다:
  - 조회 대상 주소
  - 문서 유형 (등기부등본/건축물대장)
  - 발급 일시
  - 발급 비용
- 파일명 규칙: `[문서유형]_[주소약어]_[발급일시].pdf`
  - 예: `등기부등본_중화동450_103동904호_20250207_143022.pdf`

### 5.2 부가 기능 (Should-Have / P1)

#### F6. 조회 이력 관리

- `/조회이력` 명령어로 최근 조회 내역(주소, 일시, 문서유형, 비용)을 확인할 수 있다.
- 기본 30일간의 이력을 보여주며, 기간 필터링을 지원한다.

#### F7. 월간 사용량 및 비용 리포트

- `/리포트` 명령어로 해당 월의 총 조회 건수, 총 비용, 사용자별 조회 건수를 요약 리포트로 제공한다.

#### F8. 즐겨찾기 주소 등록

- 자주 조회하는 주소를 즐겨찾기로 등록하여 단축 명령어로 조회할 수 있다.
- 예: `/등기 #중화한신904` → 등록된 주소로 자동 변환

### 5.3 향후 고려 기능 (Nice-to-Have / P2)

#### F9. 주요 변동사항 알림

- 특정 주소를 "감시 목록"에 등록하면, 등기부등본 변동(소유권 이전, 근저당 설정 등) 발생 시 슬랙 채널에 알림을 보낸다.

#### F10. 카카오톡 연동

- 슬랙 외에 카카오톡 워크스페이스에서도 동일한 기능을 지원한다.

## 8. 사용자 플로우 (User Flow)

### 8.1 정상 플로우 (Happy Path)

1. 사용자가 슬랙 `#부동산-조회` 채널에서 `/전체 서울시 중랑구 중화동 450 중화한신아파트 103동 904호` 입력
2. 봇이 "🔍 조회 중입니다... (서울시 중랑구 중화동 450 중화한신아파트 103동 904호)" 메시지 표시
3. 백엔드가 주소를 파싱하고, 등기소 API + 공공데이터 API를 병렬 호출
4. PDF 발급 완료 후, 봇이 채널에 파일 2건 업로드 + 요약 메시지 전송
5. 사용자 및 채널 멤버가 즉시 파일 확인·다운로드 가능

### 8.2 예외 플로우

| 상황 | 봇 응답 |
|------|---------|
| 주소 파싱 실패 | "❌ 주소를 인식할 수 없습니다. 다음 형식으로 입력해주세요: [시/도] [시/군/구] [읍/면/동] [번지] [건물명] [동] [호]" |
| 해당 주소 검색 결과 없음 | "❌ 해당 주소의 등기부등본을 찾을 수 없습니다. 주소를 다시 확인해주세요." |
| 동·호수 중복 매칭 | "📋 여러 건이 검색되었습니다. 아래에서 선택해주세요:" + 인터랙티브 버튼 |
| 외부 API 장애 | "⚠️ 현재 등기소 서버 연결에 문제가 있습니다. 잠시 후 다시 시도해주세요." |
| 결제 잔액 부족 | "💳 결제 잔액이 부족합니다. 관리자에게 문의해주세요." |
| 권한 없는 채널에서 호출 | "🔒 이 채널에서는 조회 기능을 사용할 수 없습니다. #부동산-조회 채널을 이용해주세요." |

## 9. 외부 연동 및 의존성 (Dependencies)

| 외부 시스템 | 용도 | 비고 |
|------------|------|------|
| **대법원 인터넷등기소** | 등기부등본 발급 | 공식 API 또는 서드파티 중개 API (예: 틸로, 바로빌 등) 활용 검토 필요 |
| **공공데이터포털 (data.go.kr)** | 건축물대장 조회 | 국토교통부 건축물대장정보 서비스 API |
| **Slack API** | 슬래시 커맨드, 파일 업로드, 인터랙티브 메시지 | Slack Bolt SDK 활용 |
| **결제 시스템** | 등기부등본 발급 비용 결제 | 선불 충전 또는 월말 정산 방식 검토 |

## 10. 데이터 모델 (핵심)

### 10.1 조회 요청 (Query Request)

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | UUID | 요청 고유 ID |
| `slack_user_id` | String | 요청자 Slack ID |
| `slack_channel_id` | String | 요청 채널 ID |
| `raw_input` | String | 원본 명령어 입력값 |
| `parsed_address` | JSON | 파싱된 주소 구성요소 |
| `doc_type` | Enum | `등기부등본` / `건축물대장` / `전체` |
| `status` | Enum | `pending` / `processing` / `completed` / `failed` |
| `file_urls` | Array | 발급된 PDF 파일 경로 |
| `cost` | Integer | 발급 비용 (원) |
| `created_at` | Timestamp | 요청 일시 |
| `completed_at` | Timestamp | 완료 일시 |

## 11. 릴리즈 계획 (Release Plan)

### Phase 1 — MVP (4주)

- 슬래시 명령어 기본 구조
- 주소 파싱 (지번 주소)
- 등기부등본 자동 조회·발급·업로드
- 기본 에러 핸들링

### Phase 2 — 기능 확장 (3주)

- 건축물대장 조회·발급·업로드
- `/전체` 명령어 (동시 조회)
- 도로명 주소 지원
- 조회 이력 관리

## 12. 리스크 및 완화 방안 (Risks & Mitigations)

| 리스크 | 영향도 | 완화 방안 |
|--------|--------|-----------|
| 인터넷등기소 공식 API 미제공 | 높음 | 서드파티 API 업체(틸로, 바로빌 등) 활용 또는 정부 API 제공 로드맵 확인 |
| 외부 API 응답 지연/장애 | 중간 | 타임아웃 설정, 재시도 로직, 장애 시 사용자 안내 메시지 |
| 개인정보 보호 이슈 | 높음 | 접근 권한 제어, 파일 암호화, 자동 삭제 정책, 개인정보처리방침 수립 |
| 등기부등본 발급 비용 누적 | 중간 | 월별 예산 상한 설정, 사용량 알림, 관리자 승인 워크플로우 |
| 주소 파싱 정확도 | 중간 | 주소 정제 라이브러리 활용 (예: Juso API), 파싱 실패 시 가이드 메시지 |

## 13. 오픈 이슈 (Open Questions)

1. **등기소 API 접근 방식**: 대법원 인터넷등기소의 직접 API 사용이 가능한지, 아니면 서드파티 중개 업체를 통해야 하는지 확인 필요
2. **결제 모델**: 건당 실시간 결제 vs. 선불 크레딧 충전 vs. 월말 일괄 정산 중 어떤 방식을 채택할 것인지
3. **데이터 보존 기간**: 발급된 PDF 파일의 서버 보존 기간 (30일 vs. 90일 vs. 사용자 설정)
4. **멀티 워크스페이스 지원**: 향후 여러 중개사무소에 SaaS 형태로 제공할 것인지
